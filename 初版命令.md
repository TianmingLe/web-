###对其中要填的内容你先用占位符（或者你自己想一些与对应主题相关的内容来填充）来代替，现在仅完成网页框架
###简历网页设计需求

一、 整体概念与情感目标

· 第一印象（0-3秒）：视频背景自动播放，氛围感拉满，但画面上元素极少，只有顶部极简导航和一个微妙的向下滚动提示（例如一道发光的向下箭头，或一行呼吸灯文字“探索我的世界”）。
· 仪式感建立：首次向下滚动时，“门扉开启”特效触发。这个特效不仅仅是一个过场，更是从“封面”进入“内容”的仪式，让浏览者感觉自己正在推开一扇通往你能力宇宙的大门。
· 气质定调：科技、清新、有分量。避免轻浮的炫技，每一个动效都要服务于“我的技术是有深度的”这一暗示。

二、 导航与状态栏

我们将其设计为固定顶部导航栏，但根据页面语境变换形态：

· 初始态（封面期）：透明背景，文字极细，带有微弱的蓝色发光。选项为你列出的：首页、能动技术、AI特种技术、自媒体特种技术、思想领域高度技术、其他。
· 门开启后：导航栏可变为半透明磨砂玻璃效果（毛玻璃），增加层次感，同时文字颜色稍微变实，使其在内容上方可读但不抢戏。
· 导航交互设想：
  · 鼠标悬停时，对应文字下方滑出一道光条，或文字笔画像被能量填充一样亮起。
  · 点击非“首页”栏目时，是否需要“关门”再“开门”到对应板块？可以设计为：点击后，页面平滑滚动到对应锚点，而门扉效果仅在首次访问首页时触发一次，或者设计一个微缩版的“内容抽屉”展开动画用于板块切换，但避免重复完整开门造成疲劳。
· 补充探讨点：你的“状态栏”称呼很特别，是否可以隐喻为“系统状态指示器”？比如每个标签旁有一个极小的光点，当前板块光点常亮，其他为暗色，强化科技感。

三、 门扉开启特效（核心技术记忆点）


· 触发与限制：
  · 严格仅首页向下滚动时触发，且仅一次（除非刷新）。
  · 需要解决“页面不往下滑动，门先打开”的矛盾：可通过劫持滚轮事件实现。用户在封面首次向下滚时，页面不滚动，而是驱动开门动画。动画结束后，滚轮控制权交还给页面正常滚动。
  · 向上滚动时门已开，直接平滑滚动回封面顶部，无再次关门动画（除非你想设计一个“离开展厅”的关门特效，但那会增加复杂度，建议保持简洁）。
· 视觉呈现：
  · 门可以是两扇无接缝的光滑金属/玻璃质感门板，或两束由粒子组成的数据流向两侧分开。

  · 门上的纹理：可以是隐约的电路板纹理、不断流动的数据线条，或者你的个人标志性符号的激光蚀刻。
· 中间内容的显露：门开启后，显露的应该是一个富有纵深感的主视觉区，然后用户继续下滚才能看到简历详情。这符合“逐步揭示”的叙事。

四、 内容区域布局与多媒体融入

页面整体采用纵向分段式长页面，每个“状态栏”对应一个大的内容区块（Section）。每个区块都应是一个多媒体体验单元。

1. 首页（封面&核心）
   · 背景：全屏循环视频，内容可以是缓慢流动的粒子云、代码雨但更抽象艺术、分子结构旋转等。需加一层深色半透明渐变或网格纹理叠加，保证前景文字清晰（但这个内容可替换）
   · 封面内容：你的名字、核心头衔（例如“能动技术探索者”），以及那个开门提示符。
   · 音乐坞（Music Dock）：从一开始就存在，但以最小化悬浮。设计成一个极简的圆形按钮，悬浮在页面右下角，鼠标悬停展开为小型播放条（播放/暂停、音量、曲目切换）。单击可控制背景音乐开关。风格应与整体科技感一致，比如半透明胶囊状。
2. 能动技术 / AI特种技术 / 自媒体特种技术 / 思想领域高度技术
   这些是你简历的核心技能展示区，每个板块建议采用统一的框架但不同色彩主题或粒子风格区分，避免杂乱。
   · 统一框架：例如，每个板块开头有一个“标题+概念简述”，随后是3D模型展示、视频案例、交互图表、图片画廊、文字阐述的有机组合。
   · 3D模型整合：提到3D模型，这是亮点。比如在“AI特种技术”板块，可以嵌入一个可拖拽旋转的神经网络结构3D模型；在“能动技术”里展示一个引擎或机械结构模型。模型应轻量，并支持鼠标拖拽旋转/缩放。
   · 视频作为内容：视频不应只作为背景。在“自媒体技术”板块，可以直接将你的作品视频放在一个仿浏览器窗口或设备框架内自动播放，无声或点击有声，周围环绕粒子。
   · 图片展示：普通的图片画廊可以用3D旋转木马或瀑布流布局，但你说希望图片随滚动3D旋转。可以实现视差滚动驱动3D旋转：图片初始为平淡角度，随着滚轮向下，图片在3D空间内绕自身轴旋转一定角度，仿佛“被看透”，增加技术感。
   · 粒子效果与鼠标交互：整个页面可设计全局微粒子系统（如悬浮的微光点），它们会被鼠标吸引或推开，形成力场效果。另外，在各板块标题或按钮上，鼠标悬停可产生粒子爆破或聚拢特效。
3. 其他板块
   · 可放置联系方式、关于我、社交媒体入口等，设计同样保持统一，但可更朴素，让用户关注信息本身。

五、 视觉风格与质感塑造（偏向苹果科技风格，清新风有质感）

· 配色方案：基底用深邃但偏蓝的灰（#0a0f1a 或类似），主色调选清透的亮青蓝或电光蓝紫渐变，点缀纯净白光。避免高饱和霓虹，采用较低饱和、高明度的“未来感清新”色彩，类似毛玻璃+无印良品式的干净布局与科技元素碰撞。
· 毛玻璃与光照：广泛运用半透明背景（backdrop-filter: blur()），配合细腻的边框发光和内阴影，模拟玻璃和金属质感。关键文字或头像周围可有柔光光晕。
· 字体：标题选用几何感强、现代的无衬线体（如Poppins, Montserrat），中文可用思源黑体或庞门正道标题体。正文清晰易读。部分技术术语可用等宽字体（如Fira Code）呈现，增强“代码”气息。
· 间距与留白：保持大留白。每个区块之间保留足够呼吸空间，让特效有施展余地，不拥挤。
· 鼠标样式：可以自定义一个极简的圆形光标或十字准星光标，并带有惯性跟随效果，强化沉浸式“操作系统”感。

六、 特效管理与“不冗杂”原则

“效果要好，同时也不要冗杂”。这是关键挑战。

· 分层触发：特效应绑定在滚动视口（Intersection Observer）。元素进入视野时才启动动画，离开则暂停或重置。避免一次性全屏粒子乱飞。
· 克制使用：全局粒子系统仅作为氛围，数量稀少。主要的华丽特效留给标题转场、3D模型、门扉和关键数据展示。其余地方多用微妙过渡，如柔和的透明度变化、轻微上移。
· 性能预算：把每一秒帧的计算量分配给最需要的地方。视频背景下，粒子数量应减少；3D模型旋转时，暂停背景视频解码或降低模型面数。设计上要考虑移动端降级。

七、 技术实现关键点探讨（仅为需求设计，非代码）

· 门扉控制：需要使用到滚动事件拦截（wheel 事件、touchmove）与状态管理（state: 'locked' | 'opening' | 'open'）。动画可用CSS 3D Transforms配合JavaScript手势驱动，或者使用更强大的WebGL库（如Three.js）完成门的3D网格和材质。
· 3D模型渲染：Three.js 几乎是必选。模型格式推荐 glTF（GLB），压缩优化。并提供Loading状态（模型位置显示一个旋转的全息线框）。需要为移动端考虑性能，可能用静态图替换3D交互。
· 视频背景与音乐：背景视频应静音自动播放，无控制器，循环。音乐坞的音频需与视频背景音量协调。用户主动播放音乐时，背景视频可保持静音。需处理浏览器的自动播放策略。
· 粒子系统：可用Canvas 2D 或 WebGL（如Particles.js的定制版，或直接用Three.js Points）。粒子需对鼠标位置产生排斥/吸引，这需要高效的向量计算。

八、 移动端与可访问性补充
· 响应式布局：3D模型在移动端缩小为可双指旋转的视图。音乐坞触屏易操作。但仍保持相同的科技感动画。
· 键盘与屏幕阅读器：虽然特效华丽，但要保留基本语义结构，为屏幕阅读器提供页面各个板块的ARIA标签，让内容可被搜索引擎和辅助技术理解。

九、 更多创意火花（一起头脑风暴）

· “思想领域高度技术”板块：可以设计成一个“脑图”或“星系”，每个思想节点是一个可点击的星球，展开文章摘要。鼠标在星系中导航，非常契合该板块调性。
· 彩蛋：在页面某处连续点击，触发一个全屏特效，比如所有粒子汇聚成你的Logo。
· 实时数据展示：如果你的“能动技术”涉及物联网或数据，可以在页面上嵌入一个模仿控制台的面板，展示模拟的实时数据图表，。
· 状态持久化：如果用户首次访问开了门，下次访问是否还要看门？可以用LocalStorage记住状态，回头客直接看到已开门的内容，但提供一个“再看一次片头”的按钮。彰显贴心设计。

### 🎬 补充模块三：关键动效与交互逻辑细化
#### 1. 门扉开启特效（Scroll Hijack 实现规范）
- **状态机**：`LOCKED` → `OPENING` → `OPEN` → `NORMAL_SCROLL`
- **劫持逻辑**：监听 `wheel` / `touchmove`，当 `state === LOCKED` 时 `e.preventDefault()`，将滚动增量映射为门板 `transform: rotateY()` 与 `opacity` 过渡。
- **动画参数**：总时长 `1.8s`，缓动函数 `cubic-bezier(0.25, 1, 0.5, 1)`。门板材质使用 `MeshPhysicalMaterial`（金属度 0.6，粗糙度 0.2，环境贴图反射）。
- **完成判定**：动画结束后写入 `localStorage.setItem('door_opened', 'true')`，解除滚动拦截，触发 `IntersectionObserver` 激活后续区块。

#### 2. 导航栏与状态指示器
- **形态切换**：封面期 `bg-transparent text-white/80`；滚动过阈值后 `backdrop-filter: blur(16px) bg-white/5 border-b border-white/10`。
- **悬停动效**：使用 `::after` 伪元素实现光条滑入（`transform: scaleX(0)→1`），或 SVG 路径描边动画（`stroke-dashoffset`）。
- **状态光点**：每个导航项右侧添加 `width: 4px; height: 4px; border-radius: 50%`，当前区块对应光点 `box-shadow: 0 0 6px #38bdf8`，滚动监听自动切换。

#### 3. 全局粒子与鼠标力场
- **实现**：Canvas 2D + 向量场计算。粒子总数 ≤ 150（移动端 ≤ 50）。
- **交互**：鼠标位置 `(mx, my)` 作为斥力源，力场半径 `120px`，粒子速度 `v += (mx - px) / r² * 0.8`。离屏粒子回收复用，避免 GC 卡顿。
- **性能保护**：当 `prefers-reduced-motion: reduce` 或设备性能等级低时，粒子数量降为 30，关闭力场计算，仅保留缓慢漂浮。

#### 4. 音乐坞（Music Dock）
- **形态**：右下角悬浮胶囊，默认 `w-10 h-10` 圆形播放图标，悬停展开为 `w-48` 控制条（播放/暂停、进度环、音量滑块、曲目名）。
- **音频策略**：背景音乐使用 `<audio loop muted>` 预加载，用户点击展开后解除 `muted`。曲目列表从 `/data/audio/playlist.json` 动态拉取。
- **状态同步**：切换路由/区块时音乐不中断；页面不可见时 `audio.pause()`，可见时恢复。

#### 5. 3D 模型与媒体容器
- **模型加载**：使用 `@react-three/drei` 的 `<Suspense>` + `<Html center>` 显示加载骨架（全息线框旋转）。
- **交互限制**：移动端禁用自动旋转，改为双指捏合缩放 + 单指拖拽；桌面端支持 `OrbitControls` 阻尼 `dampingFactor: 0.05`。
- **视频容器**：使用 `<video playsinline loop muted>` 封装在磨砂玻璃相框内，点击后触发 `requestFullscreen()` 或弹出模态框播放有声版本。

### ⚡ 补充模块四：性能预算与降级策略（强制遵守）
| 指标 | 目标值 | 实现手段 |
|------|--------|----------|
| 首屏 JS 体积 | ≤ 180KB（gzip） | 路由懒加载、`import()` 动态导入 Three.js/GSAP |
| 首屏渲染时间 | ≤ 1.5s（3G） | 视频背景使用 `<source type="video/webm">` + 降级 `<img>` 占位 |
| 动画帧率 | ≥ 55fps（桌面） / ≥ 30fps（移动） | 动画统一走 `transform`/`opacity`，禁用 `width/height/top/left` |
